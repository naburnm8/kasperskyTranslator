# Приложение "Переводчик"

Android-приложение для перевода слов с английского языка на русский с помощью [Skyeng API](https://dictionary.skyeng.ru/doc/api/external)

## Инструкции по запуску

1. Склонировать репозиторий

```pwsh
git clone https://github.com/naburnm8/kasperskyTranslator.git
```

2. Открыть проект в Android Studio или IntelliJ IDEA (с плагином Android)
3. Создать файл `local.properties` в корне проекта с указанием внутри пути до Android SDK

```properties
#Пример для Windows
sdk.dir=C\:\\Users\\username\\AppData\\Local\\Android\\Sdk
#Пример для macOS
sdk.dir=/Users/username/Library/Android/sdk
```
4. Запустить приложение (собрать и установить на эмулятор\устройство)

## Задание

Проект выполнен как тестовое задание.

Текст задания: необходимо реализовать приложение, которое переводит английские слова на русский, хранит историю переводов, поддерживает добавление в избранное и выводит избранное на отдельном экране.

## Функционал

- Перевод введенного слова с английского на русский
- Хранение истории переводов
- Добавление (добавляется при успешном переводе автоматически) и удаление слов из истории
- Хранение избранных слов
- Добавление и удаление слов из избранного
- Отдельный экран отображения избранных слов

## О реализации

### Используемые технологии

- Kotlin - основной ЯП
- Coroutines - асинхронные вызовы
- Jetpack Compose - UI
- Room - локальная БД
- Navigation - навигация между экранами
- Koin - DI
- Retrofit - работа с REST API
- ViewModel - организация MVVM

### Архитектура

Приложение построено с помощью MVVM:

1. UI (View) - Compose следит за состояниями из ViewModel 
2. ViewModel - управляет состоянием и получает данные из репозитория
3. Repository (Model) - работает с БД и API Skyeng, извлекая из них данные (или выгружая в них данные)

Также в проекте глобально задействованы два уровня сущностей: Domain и Data.

- Data-сущности - непосредственные данные получаемые от источников репозиториев
- Domain-сущности - сущности, используемые в приложении

Переход Data-Domain осуществляется в репозиториях с помощью Mapper.
